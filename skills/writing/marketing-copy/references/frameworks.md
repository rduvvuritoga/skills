# Framework and Pattern Selection

Read this reference when diagnosing a weak page, headline, problem section, feature list, or unexplained conversion problem. Choose one primary framework; combine tools only when they solve distinct problems.

## Selection

| Copy problem | Primary tool | Use it to |
|---|---|---|
| A whole page or hero feels flat | **AIDA** | Sequence attention, interest, desire, and action |
| A headline is vague or forgettable | **4Us** + five-second test | Add usefulness and specificity |
| A problem section lacks urgency | **PAS** | Name the problem, consequence, and resolution |
| Features do not connect to value | **Jobs-to-be-Done** | Translate capabilities into buyer outcomes |
| A transformation feels abstract | **Before–After–Bridge** | Make the current state, future state, and mechanism concrete |
| Conversion is weak for unclear reasons | **MECLABS** | Inspect motivation, value, friction, and anxiety |
| Brand framing makes the product the hero | **StoryBrand** | Make the customer the hero and product the guide |

## Working definitions

- **AIDA** — earn attention with a relevant problem, build interest, create desire with a believable outcome, then offer one action.
- **PAS** — state the buyer's problem in their language, show the cost of leaving it unsolved, then present the product as the resolution.
- **Jobs-to-be-Done** — express a feature as the progress the buyer hires it to create.
- **Before–After–Bridge** — show a recognizable current state, a concrete better state, and the product mechanism connecting them.
- **4Us** — make a headline useful first, then strengthen it with urgency, uniqueness, or ultra-specificity.
- **MECLABS** — test whether motivation and value outweigh friction and anxiety.
- **StoryBrand** — frame the customer as the protagonist and the product as the guide.

## Headline patterns

Adapt a pattern to the verified offer; do not paste one mechanically.

- `{Achieve outcome} without {pain}`
- `The {category} for {specific audience}`
- `{Question naming a recognizable pain}`
- `{Concrete outcome} while {common constraint}`

The headline and subhead together should identify the product, audience, and consequence. Prefer a concrete situation such as “See every project's real status without chasing five people” over a category claim such as “AI-powered project management.”

## CTA pattern

Describe the value of the next step with `[action verb] + [what they get] + [qualifier when useful]`:

- `See it in action`
- `Get my free audit`
- `Start the 14-day trial`

Use one primary CTA per conversion path. Labels such as `Submit` or `Learn more` are suitable only when they accurately describe the value the visitor receives.
