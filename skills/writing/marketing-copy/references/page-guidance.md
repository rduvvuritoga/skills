# Page and Audience Guidance

Read this reference for full-page drafting, page-type decisions, buyer-segment tailoring, or copy intended to be extracted accurately by answer engines.

## Page types

- **Homepage** — lead with the broadest true value proposition, then provide clear paths for distinct visitor intents without collapsing them into generic messaging.
- **Landing page** — match one message and CTA to the traffic source, and complete the conversion argument on the page.
- **Pricing page** — help the buyer choose, make the recommended plan legible, and answer the anxieties attached to commitment.
- **Feature page** — connect feature → benefit → outcome through a recognizable use case, then offer a clear next step.
- **About page** — connect why the company exists to a customer benefit, then provide a relevant action.

## Audience tailoring

Write to one buyer segment at a time. Lead with the pain that segment feels most acutely and the proof it trusts most.

- Hands-on or smaller buyers often value plain language, speed, and immediate payoff.
- Larger or multi-stakeholder buyers often need measured claims, rollout detail, risk reduction, and proof.

Treat these as defaults only. The project's audience research overrides them.

## AI-extractable copy

When the user wants copy that answer engines can understand or cite:

- State what the product is, who it serves, and how it differs in self-contained declarative sentences.
- Use one consistent product and category name.
- Keep factual claims specific and supported.
- Add a concise FAQ only when real buyer questions clarify the offer.
- Make the page title and meta description accurate summaries rather than keyword containers.

Keep this branch scoped to copy. Machine-readable schema, technical SEO files, and search-distribution strategy require separate implementation work.
